# Implementation Checklist

Use este checklist antes de aprovar qualquer tela ou componente.

## Design

- A tela tem um protagonista claro?
- Existe apenas uma ação principal por seção?
- O espaçamento segue a escala oficial?
- A tipografia segue a hierarquia definida?
- As cores usam tokens oficiais?
- O motion é discreto?

## Produto

- A tela responde onde o usuário está?
- A tela mostra o que aconteceu?
- A tela explica se aquilo importa?
- A tela indica o próximo passo?
- A tela mostra evidências ou contexto?

## Desenvolvimento

- Componentes são reutilizáveis?
- Tokens foram usados em vez de valores soltos?
- Estados de erro, loading e vazio foram considerados?
- Acessibilidade básica foi validada?
