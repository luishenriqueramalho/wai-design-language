# Navegação

A navegação deve refletir o que o produto entrega de forma real e consistente.

## Regra principal

Não coloque no menu uma área que ainda não está pronta ou que não tenha valor claro para o usuário.

Menu excessivo reduz confiança e deixa o produto mais difícil de entender.

## Direção para o WAI Risk

Manter navegação enxuta no MVP:

- Dashboard
- Vistorias
- Assinatura

Áreas futuras podem existir no roadmap, mas não devem aparecer como se estivessem disponíveis.

## Evitar

- Menus grandes apenas para parecer mais completo.
- Rotas sem entrega clara.
- Funcionalidades futuras expostas sem contexto.
- Nomes técnicos que o cliente não entende.

## Regra prática

Se o usuário entra em uma área e não consegue concluir uma tarefa real, o item não deve ficar visível no menu principal.
